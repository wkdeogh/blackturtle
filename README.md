# Black Turtle

미국 주식 투자 판단을 위해 시장 심리·경제 지표와 지정한 X 계정의 기업 언급·감성을 저장해 보여주는 개인용 모바일 대시보드입니다.

## 동작 원칙

- 페이지 접속은 Supabase에 저장된 마지막 성공 스냅샷만 읽습니다.
- FRED와 X는 독립 페이지이며 X 페이지에서는 유료 X 원문 수집과 저장 원문의 LLM 재분석을 따로 실행합니다.
- 갱신은 Vercel Workflow에 등록되어 브라우저 새로고침·페이지 이동·배포와 독립적으로 계속 실행됩니다.
- Supabase의 실행 상태를 폴링하므로 재접속해도 대기·수집·저장·완료·실패 상태가 복구됩니다.
- `/macro`는 CNN 공포·탐욕 지수와 FRED 경제 지표를 모바일 대응 라인 차트·해석과 함께 보여주며, `/social`은 X 수집 설정과 분석 결과를 보여줍니다.
- `/settings`에서 모니터링 계정을 추가·제거하거나 활성·비활성화하고, `/social`에서 수집 기간(1~30일)과 계정별·전체 게시물 상한을 정합니다.
- 상단 테마 버튼으로 다크·라이트 모드를 전환하며 선택값은 현재 기기에 저장됩니다.
- 기업 언급은 전체 또는 선택 계정으로 집계하고, 최근 게시물은 별도 계정 탭에서 한 계정씩 보거나 필요할 때만 전체를 펼쳐 볼 수 있습니다.
- 게시물 상한을 비우면 X API가 제공하는 범위 안에서 날짜 조건만 적용합니다.
- X 게시물의 기업 식별·감성과 전체 한국어 번역은 한 번의 OpenAI Structured Outputs 호출로 생성하며 기본 모델은 저비용 분류 모델인 `gpt-5-nano`입니다.
- 전체 게시물의 반복 주제는 별도 Structured Outputs 호출로 빈도순 요약하며 기본 모델은 `gpt-5-mini`입니다.
- X만 수집하면 새 게시물은 분석 대기로 저장되고 기존 분석은 유지됩니다. `저장 데이터 LLM 재분석`은 X API 호출 없이 저장된 전체 게시물의 기업 감성과 주제를 다시 생성하며, 골드색 통합 버튼은 X 수집 후 LLM 분석까지 연속 실행합니다.
- 성공한 갱신은 `/history`에서 실행별로 다시 볼 수 있으며, 기본 30회(설정 가능 5~100회)까지 보관합니다.
- 갱신 도중 또는 실패 시 기존 스냅샷을 계속 보여줍니다.
- 한 번 로그인한 기기는 서명된 HttpOnly 쿠키로 90일 동안 기억합니다.
- 외부 API 키와 Supabase secret key는 서버에서만 사용합니다.

## 로컬 실행

```bash
cp .env.example .env.local
npm install
npm run dev
```

`.env.local`의 값을 채운 후 Supabase SQL Editor에서
`supabase/migrations` 폴더의 SQL 파일을 번호 순서대로 실행합니다. 기존
프로젝트는 새 번호의 migration만 추가로 실행하면 됩니다.

## 검증

```bash
npm run check
```

자세한 사용자 설정 순서는 `SETUP.html`, 구현 구조는 `plan.html`에 있습니다.
