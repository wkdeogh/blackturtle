X API와 대상 계정

필수 · 비용 확인

X Developer Console에서 App을 만들고 Bearer Token을 준비합니다. 공개 계정 게시물 읽기는 앱 전용 Bearer 인증으로 설계하되, 실제 endpoint 접근 가능 여부와 사용량 과금은 신청 화면에서 최종 확인합니다.

현재 수집 규칙

  • 각 계정의 원글을 수집하고 reply와 repost는 기본 제외
  • X 모니터링 탭에서 최근 1·3·7·14·30일 중 선택
  • 계정당·전체 게시물 상한은 비워 두면 날짜 조건만 적용
  • X만 수집은 원문만 저장하고, 저장 데이터 LLM 재분석은 X API 호출 없이 저장된 게시물만 분석
  • X 수집 + LLM 분석은 설정 저장부터 두 작업까지 연속 실행
  • 보호 계정과 접근 불가 게시물은 수집 대상에서 제외

OpenAI 게시물 분석

필수 · 역할별 모델

X 게시물을 묶음으로 보내 기업별 감성을 positive | neutral | negative 구조로 받고, 같은 응답에서 게시물 전체의 자연스러운 한국어 번역도 생성합니다. 전체 게시물에서는 반복되는 주제를 빈도순으로 별도 요약합니다. 기업 분류·번역과 전체 주제 요약 모두 gpt-5.4-nano와 reasoning low를 사용합니다.

배포 후 확인

필수

준비가 끝나면

민감한 값 자체를 전달할 필요는 없습니다. Supabase migration 실행과 Vercel 환경 변수 등록을 마친 뒤 새 배포를 실행하면 첫 비밀번호 로그인과 데이터 갱신을 바로 시험할 수 있습니다.

체크 상태는 현재 브라우저의 localStorage에만 저장됩니다. API 접근 조건과 요금은 변경될 수 있으므로 실제 발급 화면을 최종 기준으로 사용하세요.
